# 悬浮小浏览器（Windows x64）

一个基于 Electron 42 与 `WebContentsView` 的独立桌面小浏览器。网页不是通过 iframe 嵌入，因此不会受网站 `X-Frame-Options` 或 CSP `frame-ancestors` 的 iframe 限制。v1.8.4 支持最多 12 个相互独立的网页标签、可收起的合并浏览栏、本地设置标签、首页书签、云游戏鼠标强制释放，以及可配置的老板键动作。

> 本项目的代码、界面、文档、测试及内置资源完全由 OpenAI Codex 生成。

## 安装与运行

### 安装版

运行 `FloatingMiniBrowser-Setup-1.8.4-x64.exe`，按向导安装。默认按当前用户安装，可选择安装目录，并创建桌面与开始菜单快捷方式。

### 便携版

直接运行 `FloatingMiniBrowser-Portable-1.8.4-x64.exe`，无需安装。设置和网站会话仍会保存在 Windows 用户应用数据目录中，因此“便携”仅指程序文件无需安装。

如需自行构建，先执行 `npm ci`，再在项目目录的 CMD 中执行 `npm.cmd run build:portable`。构建配置从 `vendor\electron` 读取 Electron 42.7.0 Windows x64 ZIP；该压缩包超过 GitHub 单文件限制，不纳入源码仓库，需将官方 `electron-v42.7.0-win32-x64.zip` 放入此目录，并用同目录的 `SHA256SUMS.txt` 校验。electron-builder 会将安装包、便携版和解包目录输出到 `build`；每次打包前会读取当前输出配置，并自动清理上一轮明确识别的构建产物。

### 自动发布

推送与 `package.json` 版本一致的标签（例如 `v1.8.4`）会触发 `.github/workflows/release.yml`。Windows runner 会下载并校验固定版本的 Electron、运行语法检查和全部测试、重新打包便携版、测试打包后的 EXE，并上传 EXE 与 SHA-256 文件到 GitHub Release。标签与包版本不一致时工作流会立即失败，避免错版发布。

> 本程序未使用代码签名证书。Windows SmartScreen 可能显示“未知发布者”；请确认文件来自可信来源后再运行。

## 使用方法

- 在地址栏输入域名或完整 URL；省略协议时自动补充 `https://`。
- 使用标签栏的 `+` 新建标签，点击切换，点击 `×` 关闭；最多同时打开 12 个标签页。
- 设置以独立本地标签打开，可像网页标签一样切换、关闭，且不占用网页标签数量。
- 首页内置抖音、知乎、哔哩哔哩、百度和微博入口；可从设置标签导入标准浏览器书签 HTML，最多 100 个，内置入口始终保留。
- 支持 `Ctrl+T` 新建、`Ctrl+W` 关闭当前标签、`Ctrl+Tab` / `Ctrl+Shift+Tab` 切换标签。
- 各标签页分别保留导航历史、滚动位置和页面表单状态；退出后恢复标签 URL 与当前标签。
- 网站通过 `target="_blank"` 或 `window.open()` 打开的普通 HTTP/HTTPS GET 页面会进入应用内新标签，不再弹出独立窗口。
- 云游戏请求 Pointer Lock 时，仅当前可见标签会获准。鼠标被游戏接管后，按全局快捷键 `Alt+Shift+M` 强制释放；也可以按 `Esc` 或使用标题栏的“↖”按钮。
- 切换标签、打开设置、按老板键、最小化、隐藏到托盘或窗口失去焦点时都会自动释放鼠标。
- 仅允许打开 `http://` 和 `https://` 网页。
- 标签页和后退、前进、刷新、地址栏、外部打开及设置操作合并在同一行；标题栏的收起按钮可直接隐藏或恢复整行，网页区域会同步扩展，并记住选择。
- 可拖动标题栏移动窗口，拖动窗口边缘原生缩放；最小尺寸为 `360×320`。
- 默认窗口为 `480×680`，距屏幕右下角 24 像素，90% 不透明并保持置顶。
- 关闭按钮默认将程序收进系统托盘；可在设置中改为“完全退出”。该选项同时作用于普通窗口和广告画面的关闭按钮。
- 设置页可以调整透明度、置顶、开机启动、老板键和广告图片。
- 可开启“移出窗口后完全透明”；启用时强制保持置顶，鼠标离开网页窗口区域时透明度降为 0%，移回后恢复离开前的透明度。广告模式不应用置顶和移出透明设置。

## 老板键与广告模式

- 默认全局老板键为 `Alt+Shift+A`，即使窗口失去焦点也能触发。
- 老板键动作可选“显示广告画面”或“静音所有网页并隐藏窗口”，默认显示广告；再次按下会恢复按键前的窗口与音频状态。
- 广告模式会将窗口调整到屏幕右下角，默认尺寸为 `440×586`，并支持在 `200×200` 至 `4096×4096` 范围内自定义宽高；同时切换为完全不透明、静音所有网页，并显示模拟广告关闭按钮。点击该按钮会按配置隐藏到托盘或完全退出，不会恢复网页。
- “静音并隐藏”不会进入广告模式或显示广告界面，只会立即静音所有网页并隐藏主窗口和网页子窗口。
- 退出广告模式后会恢复切换前的窗口位置、尺寸、透明度及各网页原有的静音状态。
- 如果原窗口隐藏在托盘或已最小化，第一次按下会弹出广告；第二次按下恢复隐藏或最小化状态。
- 自定义老板键必须包含至少一个 `Control`、`Alt` 或 `Shift` 修饰键和一个普通按键。注册冲突时旧快捷键继续有效。
- 内置 10 张原创复古游戏风广告，每 3.5 秒循环切换；进入广告模式时从第 1 张开始。
- 自定义广告支持一次多选 PNG、JPEG 或 WebP，最多 20 张、每张最大 2 MB；按选择顺序循环播放，再次选择会替换整组图片。
- 自定义图片复制到本机应用数据目录，不会交给远程网页；删除操作会删除整组自定义图片并回退到内置广告。

## 数据、权限与安全边界

- 网站 Cookie、缓存和登录状态使用持久化 Electron 会话保存在本机。
- 应用设置使用 JSON 文件保存在 Electron 的用户数据目录，包括窗口位置、透明度、置顶、老板键动作、开机启动、书签、最后 URL 和广告配置。
- 不收集浏览记录，不上传 URL、Cookie 或广告图片，不使用远程服务。
- 远程网页启用 `sandbox` 与上下文隔离，关闭 Node 集成，并保持 `webSecurity` 开启。
- 仅当前可见的 HTTP/HTTPS 标签可申请 Pointer Lock；摄像头、麦克风、定位、通知及其他敏感网站权限统一拒绝。
- TLS 证书错误不绕过；`file:`、`javascript:`、`data:`、浏览器内部协议及其他非 HTTP/HTTPS 协议不会由网页自动启动。
- HTTP/HTTPS 弹窗使用同一持久化会话打开受限子窗口；切换广告时子窗口会暂时隐藏。
- 下载会弹出 Windows“另存为”对话框；取消选择会终止下载。

## 已知边界

- 不提供书签跨设备同步、浏览器扩展或开发者工具入口；导入书签仅保存在本机，标签恢复仅保存 URL，不恢复网页进程内的临时表单内容。
- 拒绝网站敏感权限是固定安全策略，依赖摄像头、麦克风、定位或通知的网站功能不可用。
- 某些网站可能因反自动化、DRM、企业证书、第三方 Cookie 或仅支持特定浏览器而不能正常工作。
- `about:blank`、POST 提交或依赖 `window.opener` 的特殊登录流程仍可能使用受限子窗口；强行改为标签会破坏其页面上下文。
- 应用不绕过登录、访问控制、证书错误或网站自身的安全策略。

## 从源码运行

需要 Windows x64、Node.js 20 或更高版本：

```powershell
npm install
npm start
```

验证与打包：

```powershell
npm run check
npm test
npm run test:integration
npm run build:win
```

依赖已精确固定为 Electron `42.7.0` 与 electron-builder `26.15.3`，实际安装版本记录在 `package-lock.json`。
